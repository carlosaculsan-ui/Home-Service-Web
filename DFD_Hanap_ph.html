Data Flow Diagram — Level 2

Data Dictionary

External Entities

EntityDescription
CustomerA registered user who searches for and books home services. Can manage bookings, use E-Wallet, leave reviews, and chat with taskers.
TaskerA vetted service professional (electrician, plumber, cleaner, etc.) who accepts and fulfills job bookings. Earns via wallet and can cash out.
AdminPlatform administrator. Approves tasker applications, manages bookings, processes refunds, moderates reviews, and monitors analytics.
PayMongoThird-party payment gateway that processes GCash, PayMaya, and Credit/Debit Card transactions via REST API.
SupabaseBackend-as-a-service providing PostgreSQL database, user authentication, real-time subscriptions, and file storage.
Google OAuthThird-party login provider. Supplies verified user identity to the system for account creation or sign-in.
Groq AIAI API that analyzes customer task descriptions for relevance and completeness. Returns a confidence score to guide the user.

Data Stores

StoreDescriptionKey Fields
D1 — profiles Stores all user account data for customers, taskers, and admins. id, role, full_name, email, phone, wallet_balance, is_archived, created_at
D2 — bookings Records all job bookings including service details, pricing, payment info, and status lifecycle. id, client_id, tasker_id, service, status, scheduled_date, estimated_total, tasker_payout, platform_fee, helper_fee, paymongo_payment_id, reference_number, is_refunded
D3 — reviews Stores post-completion customer reviews with star ratings, text, and photo uploads. id, booking_id, tasker_id, customer_id, rating, comment, images, is_flagged, is_hidden
D4 — notifications Stores in-app alerts generated by booking updates, payments, and messages. id, user_id, title, message, is_read, created_at
D5 — wallet_transactions Financial ledger of all wallet debits and credits — payments, refunds, earnings, withdrawals. id, user_id, booking_id, amount, type (debit/credit), description, created_at
D6 — messages In-app chat messages between customers and taskers, linked to specific bookings. id, booking_id, sender_id, receiver_id, message_text, is_read, created_at
D7 — taskers Tasker professional profiles including availability, hourly rate, status, and application documents. id, user_id, name, status (pending/approved/rejected), hourly_rate, availability, is_available, service_area
D8 — tasker_leaves Approved leave dates per tasker used to determine availability during booking scheduling. id, tasker_id, leave_dates (JSON array), status (pending/approved/rejected)
